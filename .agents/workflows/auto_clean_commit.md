# 開発環境の自動クリーンアップ・ルール (Auto-Clean & Atomic Commits)

このワークフローは、Neo+の開発においてソース管理を常にクリーンに保つための恒久的ルールです。各タスク・Stepの完了時に必ず実行してください。

## 1. Atomic Commits（自動コミットの徹底）
タスクや特定の改修（例: 「Home画面の分離」「X機能の追加」など）が完了した際、ユーザー（CEO）からの指示を待たずに、以下の確認とコミットを自律的に行ってください。

1. **不要ファイルの検証**: `git status` を確認し、そのタスクに関係のないファイル（デバッグ用スクリプト、一時ファイルなど）がステージングに含まれていないか確認する。関係ないものは `git restore` 等で破棄・除外する。
2. **意味のある単位の確認**: 変更内容が1つの論理的・意味のある単位（機能追加、バグ修正、リファクタリング等）にまとまっているか確認する。
3. **コミットの実行**: 問題なければ、自律的に `git add` と `git commit -m "feat/fix/chore: ..."` を実行する。
4. **リセットの確認**: 操作後、未コミットの変更が「0件」になったことを確認してから、次の作業フェーズやユーザーへの報告に移る。

## 2. 異常検知（Anomaly Detection）
`git status` を確認した際、**変更件数が50件を超えている場合**は、以下の緊急アクションを取ること。

1. 直ちに作業を一時停止する。
2. これ以上変更を加えず、自動コミットも行わない。
3. `notify_user` ツールを使用し、「変更が50件を超えています。整理フェーズ（Gitクリーンアップ）を挟むべきか指示をください」とCEOに報告・判断を仰ぐ。

## 3. Auto-Ignore（恒久的ルール）
プロジェクトルートの `.gitignore` には以下のシステム/ビルドファイルが常に含まれていること。誤って管理対象にしないよう注意する。
`node_modules/`, `.DS_Store`, `dist/`, `build/`, `.env`
